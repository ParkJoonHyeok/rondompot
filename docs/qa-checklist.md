# RandomPot QA Checklist (1st)

배포 전/후 반복 확인용 수동 테스트 체크리스트입니다.  
목표는 기능 추가가 아니라 현재 구현 상태의 안정성 확인입니다.

## 0) 기본 환경
- [ ] 프론트 배포 URL 접근 가능
- [ ] 백엔드 헬스체크(`/actuator/health`) 200 확인
- [ ] 브라우저 캐시/세션 영향 확인이 필요하면 새 시크릿 창에서도 재확인

---

## 1) 홈

### 정상 케이스
- [ ] 홈 진입 시 헤드라인/설명/빠른 시작 카드 4개가 정상 노출된다.
- [ ] 빠른 시작 CTA가 각각 올바른 라우트로 이동한다. (`/roulette`, `/punishment`, `/mission`, `/team`)

### 최근 사용 기능
- [ ] 최근 사용 기능 값이 없으면 섹션이 노출되지 않는다.
- [ ] 기능 페이지 방문 후 홈 복귀 시 최근 사용 기능 섹션이 노출된다.
- [ ] "바로 이어서 시작하기"가 최근 사용 기능 페이지로 정확히 이동한다.

### 모바일/회귀
- [ ] 360px 폭에서 가로 스크롤 없이 카드/CTA가 깨지지 않는다.
- [ ] 버튼 터치 영역이 충분하고 오탭이 과도하지 않다.

---

## 2) 룰렛

### 정상 케이스
- [ ] 제목/항목(2개 이상) 입력 후 실행 시 결과가 표시된다.
- [ ] 결과 카드에 복사/공유(지원 브라우저) 버튼이 노출된다.

### 실패 케이스
- [ ] 항목 1개 입력 시 검증 에러가 표시된다.
- [ ] 에러 발생 시 기존 입력값은 유지된다.

### 로딩/중복 클릭
- [ ] 실행 중 버튼이 disabled 상태로 바뀐다.
- [ ] 실행 중 연타해도 중복 요청이 발생하지 않는다.
- [ ] 로딩 문구가 "룰렛 돌리는 중..."으로 표시된다.

### 복사/공유/세션/히스토리
- [ ] 결과 복사 시 텍스트 포맷이 의도대로 붙여넣기 된다.
- [ ] 공유 지원 브라우저에서 공유 시트가 열린다.
- [ ] 제목/항목 입력값이 새로고침 후 sessionStorage에서 복원된다.
- [ ] 결과 성공 시 최근 결과가 히스토리에 추가된다. (최신 위, 최대 5개)
- [ ] 히스토리 항목 복사/공유/삭제 동작이 정상이다.
- [ ] 히스토리 "전체 비우기"가 정상 동작한다.

### 모바일/회귀
- [ ] 360px 폭에서 폼/결과/히스토리 영역 가로 스크롤이 없다.

## 5) 팀나누기

### 정상 케이스
- [ ] 참가자(2명 이상) + 팀 수(2 이상, 참가자 수 이하) 입력 시 결과가 표시된다.
- [ ] 팀별 결과가 줄바꿈/가독성 깨짐 없이 보인다.

### 실패 케이스 (특화)
- [ ] 참가자 1명 입력 시 검증 에러가 표시된다.
- [ ] 팀 수 1 입력 시 검증 에러가 표시된다.
- [ ] 팀 수 > 참가자 수 입력 시 검증 에러가 표시된다.

### 로딩/중복 클릭
- [ ] 실행 중 버튼 disabled 및 "팀 나누는 중..." 문구가 표시된다.
- [ ] 실행 중 연타해도 중복 요청이 발생하지 않는다.

### 복사/공유/세션/히스토리
- [ ] 결과 복사 텍스트에 팀별 줄바꿈 포맷이 유지된다.
- [ ] 공유 지원 브라우저에서 공유가 정상 동작한다.
- [ ] 제목/참가자/팀 수 입력값이 sessionStorage에서 복원된다.
- [ ] 히스토리 추가/복사/공유/삭제/전체 비우기가 정상이다.

### 모바일/회귀
- [ ] 360px 폭에서 팀 결과 텍스트가 카드 밖으로 넘치지 않는다.
- [ ] 폼/결과/히스토리 전체에서 가로 스크롤이 없다.

## 6) 공통 운영 품질

### 접근성(a11y) 기본
- [ ] 라벨 클릭 시 해당 입력 요소에 포커스가 이동한다.
- [ ] 에러/결과/피드백이 라이브 영역에서 자연스럽게 인지된다.
- [ ] 로딩 중 폼 영역 `aria-busy` 동작이 유지된다.

### 라우팅/에러 페이지
- [ ] 존재하지 않는 경로 접근 시 `not-found` 페이지가 노출된다.
- [ ] `not-found`의 홈/기능 바로가기 링크가 정상 이동한다.
- [ ] 런타임 에러 발생 시 `error.tsx`가 노출된다.
- [ ] `error.tsx`의 "다시 시도" / "홈으로 가기"가 정상 동작한다.

### 메타/공유/검색 엔진
- [ ] 브라우저 탭 title/description 메타가 의도대로 반영된다.
- [ ] OG/Twitter 메타와 기본 썸네일(`og-image.svg`)이 노출된다.
- [ ] `robots.txt` 접근 가능, sitemap URL이 표시된다.
- [ ] `sitemap.xml`에 `/, /roulette, /punishment, /mission, /team`가 포함된다.

### PWA 기본
- [ ] `manifest.webmanifest` 접근 가능
- [ ] 모바일에서 홈 화면 추가 시 앱 이름/아이콘/테마 컬러가 기본 적용된다.
